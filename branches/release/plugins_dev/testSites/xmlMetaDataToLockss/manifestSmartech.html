This collection consists of MetaData (XML files) and document files referenced in the MetaData. A stylesheet is used on the server side to transform the XML data into HTML with links to document files, so that a LOCKSS crawl guided by a plugin can discover these document links. A corresponding LOCKSS plugin should contain rules that INCLUDE the xml (metadata) and the documents themselves.

This particular example shows how to use the OAI export at http://smartech.gatech.edu, Georgia Tech's DSPACE based institutional repository. This manifest page includes links to dynamically generated DublinCore XML of particular record sets. The style sheet and script on the the webserver cooperate to transform the generated Dublin Core XML to HTML with links to DSPACE item pages that end in '?mode=full'. Including this parameter in links enables us to define a simple plugin that has the usual base_url parameter and a hdl_id parameter. The folllowing rules create the desired behaviour of including metadata and docuemt files in a crawl avoiding the inclusion of DSPACE collection and community pages.

Please find more documentation at MetaWiki: HarvestDSPACESites

To use this approach to prepare a DSPACE/XML based collection for preservation with LOCKSS/MetaArchice it is easiest to place the stylesheet, transformer script, and manifest page on the same server as the content that is to be preserved.

The Details: Script, StyleSheet ...

  • transform.php, php script that applies the stylesheet
  • smartech-oai.xsl style sheet that converts DublinCore XML to html
  • http://www.eprints.org/software/xslt.php style stylesheet developed for eprints, which we used as template
