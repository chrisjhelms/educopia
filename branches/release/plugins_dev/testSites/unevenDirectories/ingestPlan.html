Uneven Collection - Evenly Sized Archival Units

Manifest Page to Allow preservation by LOCKSS daemons in the MetaArchive Network
LOCKSS system has permission to collect, preserve, and serve this Archival Unit.

Assuming that content in 2009 and 2010 subdirectories will not significantly change and assuming the following fictitious sizes: 

 
 5GB	2009/Jan
10GB	2009/Feb
12GB	2009/Mar
.5GB	2010/hardCoverFiction
12GB	2010/hardCoverNonFiction
20GB	2010/paperbackTrade

we will have to come up with a plan by which we ingest the small 2010/hardCoverFiction directory with other content, since at .5GB it is too small to warrant the creation of a separate archival unit. While not necessary it would be nice to combine some of the other directories into archival units as well.

One approach is

  1. to create shadow directories to combine some of the smaller subdirectories
  2. place links to the included subdirectories inside the shadow directories
  3. and hand the names of the showdow directories to a plugin
  4. which must be configured to use a directory name as parameter

In the fictitous case we might

  • create 2009/Jan_Feb_Shadow/
  • leave 2009/Mar
  • create 2010/hardCoverFictionNonFiction_Shadow
  • leave 2010/paperbackTrade

Now all we need is a plugin

  • that uses a year and a volume parameter
  • that starts at a manifest page which points to the root directory root directory of the collection
  • and then proceeds info the subdirectory for the given year
  • then into the the subsubdirectory for the given volume
  • and includes everuything inside
In this example we will configure four archival units
  • year=2009 volume=Jan_Feb_Shadow
  • year=2009 volume=Mar
  • year=2010 volume=hardCoverFictionNonFiction_Shadow
  • year=2010 volume=paperbackTrade
